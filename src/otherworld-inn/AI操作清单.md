# AI 操作清单

这份清单给 AI 玩家使用。人类玩家使用网页前端，AI 玩家使用 CLI；如果通过 MCP 连接，则 MCP 只是把文字传给 CLI，再把 CLI 输出传回来。

## 基本规则

- 你是玩家B。
- 不要操作网页前端。
- 不要直接调用游戏后端 API。
- 使用 CLI 指令和游戏交互。
- 如果通过 MCP 使用，只调用两个工具：`output` 和 `input`。
- `output` 负责发送一行 CLI 指令。
- `input` 负责读取 CLI 返回的文字。

## MCP 使用方式

每次操作通常分两步：

```text
output("你的 CLI 指令")
input()
```

例子：

```text
output("look")
input()
```

注意：

- `output` 里不要写 `node cli.js`。
- 只写 CLI 指令本身，例如 `look`、`say ...`、`hide ...`。
- `input()` 读完会清空未读输出。
- 如果刚发送了行动，务必调用 `input()` 查看世界回复。

## 常用流程

### 1. 查看有哪些游戏

```text
output("list")
input()
```

用途：列出进行中的游戏和已结算的游戏。

### 2. 加入一局游戏

```text
output("join <sessionId>")
input()
```

例子：

```text
output("join rp_1234567890_abcd12")
input()
```

用途：选择当前 AI 要参与的游戏。

### 3. 查看当前局面

```text
output("look")
input()
```

用途：查看最近场景、最近对话、我的公开状态、对方公开状态。

这是最常用的读取指令。行动前通常先 `look`。

### 4. 发送公开消息

```text
output("say 公开发言或公开行动")
input()
```

例子：

```text
output("say 我看向门牌，确认上面写的房间号。")
input()
```

用途：发送双方都能看到的内容。

适合：

- 说话
- 公开动作
- 与场景互动
- 让对方知道你正在做什么

### 5. 发送隐藏行动

```text
output("hide 隐藏行动")
input()
```

例子：

```text
output("hide 趁对方看门牌时，我检查抽屉夹层。")
input()
```

用途：发送只有你和世界引擎知道的行动。

适合：

- 偷看
- 偷藏
- 暗中准备
- 隐瞒意图
- 不想让对方知道的动作

世界引擎会判断隐藏行为是否成功，以及是否留下痕迹。

### 6. 同时发送公开和隐藏

```text
output("both 公开内容|隐藏内容")
input()
```

例子：

```text
output("both 我把手电筒照向走廊尽头。|同时把刚捡到的钥匙藏进袖口。")
input()
```

用途：一轮里同时做公开行为和隐藏行为。

注意：

- 中间必须用 `|` 分隔。
- `|` 左边是公开内容。
- `|` 右边是隐藏内容。

## 信息查询指令

### 查看任务

```text
output("quest")
input()
```

用途：查看自己的主线任务和隐藏任务。

### 查看背包

```text
output("bag")
input()
```

用途：查看自己的背包，以及你知道的对方公开背包。

提示：背包可能不是实时更新的。行动后可以重新查看。

### 查看世界设定

```text
output("world")
input()
```

用途：查看当前世界观和基础规则。

### 查看对方公开状态

```text
output("other")
input()
```

用途：查看对方的外貌、着装、手持物、身体状态和你知道的公开背包。

### 查看最近日志

```text
output("log")
input()
```

用途：查看最近几轮的公开输入和世界回复摘要。

## 游戏管理指令

### 新建游戏

```text
output("new <主题>")
input()
```

例子：

```text
output("new 搞怪")
input()
```

用途：创建新游戏。

可用主题通常包括：

- `休闲`
- `刺激`
- `恐怖`
- `搞怪`
- `梦幻`
- `风格化`

新建后世界生成需要等待。过一会儿再：

```text
output("look")
input()
```

### 结算当前游戏

```text
output("end")
input()
```

用途：归档并结算当前游戏。

不要随便使用。通常需要人类玩家同意。

### 进入度假模式

```text
output("vacation")
input()
```

用途：停止骰子和事件，让当前世界进入自由闲逛状态。

也可以从归档恢复：

```text
output("vacation <archiveId>")
input()
```

## 行动建议

- 行动前先 `look`。
- 不确定自己有什么任务时用 `quest`。
- 不确定自己带着什么时用 `bag`。
- 不确定世界规则时用 `world`。
- 不确定对方公开状态时用 `other`。
- 每次 `say`、`hide`、`both` 后都要 `input()`。
- 公开内容不要替对方说话或替对方行动。
- 隐藏行动也要写成自己尝试做什么，不要直接写结果。

好例子：

```text
output("say 我把桌上的信封推到灯光下，看看封口有没有拆过。")
input()
```

```text
output("hide 我记下信封背面的编号，但暂时不告诉对方。")
input()
```

不推荐：

```text
output("say 我发现真相并让对方震惊。")
input()
```

原因：这替世界和对方决定了结果。
